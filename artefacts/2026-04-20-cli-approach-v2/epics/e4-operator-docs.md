## Epic: Operator-facing documentation and integration guide

**Discovery reference:** `artefacts/2026-04-20-cli-approach-v2/discovery.md`
**Benefit-metric reference:** `artefacts/2026-04-20-cli-approach-v2/benefit-metric.md`
**Slicing strategy:** Risk-first

## Goal

A platform operator adopting the CLI for the first time on a new project can complete install + first feature cycle without engineering assistance or session-specific prior context. The integration guide names the commands, the expected flow, the first-run friction points observed during MVP validation (Epic 3), and the workflow-declaration authoring runbook for operators adding a skill to their local declaration post-MVP. Documentation is grounded in observed first-run experience (Epic 3), not speculative.

## Out of Scope

- API reference documentation auto-generated from the code — generated by tooling, not authored here.
- Marketing / overview materials — handled at platform level, not feature level.
- Video / screencast content — not an MVP format.
- Documentation for the adapter internals — owned by `p4-enf-cli` story artefacts and source comments.
